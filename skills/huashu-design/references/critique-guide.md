# 設計評審深度指南

> Phase 7 的詳細參考。提供評分標準、場景側重點、常見問題清單。

---

## 評分標準詳解

### 1. 哲學一致性（Philosophy Alignment）

| 分數 | 標準 |
|------|------|
| 9-10 | 設計完美體現了選定哲學的核心精神，每個細節都有哲學依據 |
| 7-8 | 整體方向正確，核心特徵到位，個別細節偏離 |
| 5-6 | 能看出意圖，但執行時混入了其他風格元素，不夠純粹 |
| 3-4 | 僅在表面模仿，未理解哲學內核 |
| 1-2 | 與選定哲學基本無關 |

**評審要點**：
- 是否使用了該設計師/機構的標誌性手法？
- 色彩、字體、佈局是否符合該哲學體系？
- 有沒有「自相矛盾」的元素？（如選了 Kenya Hara 卻塞滿內容）

### 2. 視覺層級（Visual Hierarchy）

| 分數 | 標準 |
|------|------|
| 9-10 | 使用者（User）視線自然沿設計者意圖流動，訊息獲取零摩擦 |
| 7-8 | 主次關係清晰，偶有 1-2 處層級模糊 |
| 5-6 | 能分出標題和內文，但中間層級混亂 |
| 3-4 | 訊息平鋪，沒有明確的視覺入口 |
| 1-2 | 混亂，使用者不知道先看哪裡 |

**評審要點**：
- 標題與內文的字號對比是否足夠？（至少 2.5 倍）
- 顏色/粗細/大小是否建立了 3-4 個清晰層級？
- 留白是否在引導視線？
- 「瞇眼測試」：瞇起眼看，層級是否仍然清晰？

### 3. 細節執行（Craft Quality）

| 分數 | 標準 |
|------|------|
| 9-10 | 像素級精確，對齊、間距、顏色無任何瑕疵 |
| 7-8 | 整體精緻，有 1-2 處微小對齊/間距問題 |
| 5-6 | 基本對齊，但間距不統一，顏色使用不夠系統 |
| 3-4 | 明顯的對齊錯誤、間距混亂、顏色過多 |
| 1-2 | 粗糙，看起來像草稿 |

**評審要點**：
- 是否使用了統一的間距系統（如 8pt 網格）？
- 同類元素的間距是否一致？
- 顏色數量是否受控？（通常不超過 3-4 種）
- 字體家族是否統一？（通常不超過 2 種）
- 邊緣對齊是否精確？

### 4. 功能性（Functionality）

| 分數 | 標準 |
|------|------|
| 9-10 | 每個設計元素都服務於目標，零冗餘 |
| 7-8 | 功能導向明確，有少量可刪減的裝飾 |
| 5-6 | 基本可用，但有明顯的裝飾性元素分散注意力 |
| 3-4 | 形式大於功能，使用者需要努力尋找訊息 |
| 1-2 | 完全被裝飾淹沒，失去了傳達訊息的能力 |

**評審要點**：
- 刪掉任何一個元素，設計會變差嗎？（如果不會，就應該刪）
- CTA / 關鍵訊息是否在最顯眼的位置？
- 是否有「因為好看所以加上去」的元素？
- 訊息密度與載體是否匹配？（PPT 不宜太密，PDF 可以更密）

### 5. 創新性（Originality）

| 分數 | 標準 |
|------|------|
| 9-10 | 令人耳目一新，在該哲學框架內找到了獨特表達 |
| 7-8 | 有自己的想法，不是簡單的模板套用 |
| 5-6 | 中規中矩，看起來像模板 |
| 3-4 | 大量使用了 cliché（陳腔濫調，如漸變圓球代表 AI） |
| 1-2 | 完全是模板或素材拼湊 |

**評審要點**：
- 是否避免了常見 cliché？（見下方「常見問題清單」）
- 在遵循設計哲學的同時是否有個人表達？
- 是否有「意想不到但很合理」的設計決策？

## 場景評審側重

不同輸出類型的評審重點不同：

| 場景 | 最重要維度 | 次重要 | 可放寬 |
|------|-----------|--------|--------|
| 微信公眾號封面/配圖 | 創新性、視覺層級 | 哲學一致性 | 功能性（單圖不涉及互動） |
| 資訊圖表 | 功能性、視覺層級 | 細節執行 | 創新性（準確優先） |
| PPT / Keynote | 視覺層級、功能性 | 細節執行 | 創新性（清晰優先） |
| PDF / 白皮書 | 細節執行、功能性 | 視覺層級 | 創新性（專業優先） |
| 落地頁（Landing Page） / 官網 | 功能性、視覺層級 | 創新性 | —（全面要求） |
| App UI | 功能性、細節執行 | 視覺層級 | 哲學一致性（可用性優先） |
| 小紅書配圖 | 創新性、視覺層級 | 哲學一致性 | 細節執行（氛圍優先） |

## 常見設計問題 Top 10

### 1. AI 科技 cliché
**問題**：漸變圓球、數字雨、藍色電路板、機器人臉
**為什麼是問題**：使用者已經對這些視覺疲勞，無法區分你和其他人
**修復**：用抽象隱喻替代直白符號（如用「對話」的隱喻而非聊天氣泡圖示）

### 2. 字號層級不足
**問題**：標題和內文差距太小（< 2.5 倍）
**為什麼是問題**：使用者無法快速定位關鍵訊息
**修復**：標題至少為內文的 3 倍（如內文 16px → 標題 48-64px）

### 3. 顏色過多
**問題**：使用 5 種以上顏色，沒有主次
**為什麼是問題**：視覺混亂，品牌感弱
**修復**：限制為 1 個主色 + 1 個輔色 + 1 個強調色 + 灰階

### 4. 間距不統一
**問題**：元素間距隨意，沒有系統
**為什麼是問題**：看起來不專業，視覺節奏混亂
**修復**：建立 8pt 網格系統（間距只用 8/16/24/32/48/64px）

### 5. 留白不足
**問題**：所有空間都被內容填滿
**為什麼是問題**：訊息擁擠導致閱讀疲勞，反而降低訊息傳達效率
**修復**：留白至少佔總面積 40%（極簡風格 60%+）

### 6. 字體過多
**問題**：使用 3 種以上字體
**為什麼是問題**：視覺噪音，削弱統一感
**修復**：最多 2 種字體（1 種標題 + 1 種內文），用字重和大小創造變化

### 7. 對齊不一致
**問題**：有的左對齊，有的居中，有的右對齊
**為什麼是問題**：破壞視覺秩序感
**修復**：選定一種對齊方式（推薦左對齊），全域統一

### 8. 裝飾大於內容
**問題**：背景圖案/漸變/陰影搶了主要內容的風頭
**為什麼是問題**：本末倒置，使用者來看訊息不是看裝飾
**修復**：「如果刪掉這個裝飾，設計會變差嗎？」如果不會，就刪

### 9. 賽博霓虹濫用
**問題**：深藍底(#0D1117) + 霓虹色發光效果
**為什麼是問題**：預設（Default）審美禁區（本 skill 的品位基線），且已成為最大 cliché 之一——使用者可按自己品牌 override（覆蓋）
**修復**：選擇更有辨識度的配色方案（參考 20 種風格的色彩系統）

### 10. 訊息密度與載體不匹配
**問題**：PPT 裡放了一整頁文字 / 封面圖裡塞了 10 個元素
**為什麼是問題**：不同載體的最佳訊息密度不同
**修復**：
- PPT：每頁 1 個核心觀點
- 封面圖：1 個視覺焦點
- 資訊圖表：分層展示
- PDF：可以更密，但需要清晰的導覽
