# Anti Slop：反 AI Slop 指南

AI 預設會往視覺最大公約數靠。這份文件的目標是保住品牌識別度，不是做審美潔癖。

## 為什麼要反 slop

1. 使用者要的是品牌被認出來
2. AI 預設產出是訓練語料平均值
3. 平均值會把品牌磨平
4. 所以反 slop 是保護品牌，不是單純嫌東西俗

## 高風險元素

| 元素 | 為什麼危險 | 什麼情況可用 |
|---|---|---|
| 激進紫色漸變 | AI 科技感萬用模板 | 品牌本身就這樣 |
| Emoji 圖示 | 不專業時最容易拿來湊 | 品牌或受眾本來就輕鬆 |
| 圓角卡片 + 左彩色 border | 2020-2024 常見套版味 | 使用者明確要求 |
| SVG 畫人物或場景 | 一眼就是 AI 味廉價圖 | 幾乎不要 |
| CSS 剪影代替真產品 | 最終會變 generic 科技動畫 | 幾乎不要 |
| Inter / Roboto / Arial 作 display | 識別度太低 | 品牌規範明寫 |
| 深藍黑底賽博霓虹 | 太像 GitHub dark mode 衍生物 | 品牌本身就走這路線 |

## 正向原則

- 用真實品牌資產，先讀 `references/asset-protocol.md`
- 用 `oklch()` 或品牌既有色，不現場亂發明
- 用 `text-wrap: pretty`、CSS Grid、精準排版細節
- 中文文案用「」
- 一個細節做到 120%，其他做到 80%

## 反例如何隔離

當任務本身要展示 bad example，不要把整頁做爛。用虛線邊框、角標、說明文字把反例框起來，讓它服務於敘事。
